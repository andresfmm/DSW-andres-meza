Hojas de Estilo en Cascada CSS


Hojas de Estilo en Cascada (del inglés Cascading Style Sheets) o CSS es el lenguaje de estilos utilizado para describir la presentación de documentos HTML o XML (en-US) (incluyendo varios languages basados en XML como SVG, MathML o XHTML). CSS describe como debe ser renderizado el elemento estructurado en la pantalla, en papel, en el habla o en otros medios.

CSS es uno de los lenguajes base de la Open Web y posee una especificación estandarizada por parte del W3C. Anteriormente , el desarrollo de varias partes de las especificaciones de CSS era realizado de manera sincrónica, lo que permiía el versionado de las recomendaciones. Probablemente habrás escuchado acerca de CSS1, CSS2.1, CSS3. Sin embargo, CSS4 nunca se ha lanzado como una versión oficial.

Desde CSS3, el alcance de las especificaciones se incrementó de forma significativa y el progreso de los diferentes modulos de CSS comenzó a mostrar varias diferencias, lo que hizo más efectivo desarrollar y publicar recomendaciones separadas por módulos. En vez de versionar las especificaciones de CSS, la W3C actualmente realiza una captura de las ultimas espcificaciones estables de CSS.

Hojas de Estilo en Cascada CSS

Cascading Style Sheets son las siglas en inglés para el css, es un lenguaje para diseño grafico que se implementa en las páginas web y móviles con react native vue native entre otros, para dispositivos móviles, se usa para que el documento tenga colores tipografia y estilos bonitos que sean interfaces graficas agradables para el usuario, no tiene funcionalidad es solo para la presentación y termina en extensión .css y su especificación Es MIME type text/css mantenida por mantenida por el World Wide Web Consortium (W3C) aunque los estilos se pueden poner en la misma pagina es preferible usar archivos externos y enlazarlos por ejemplo

De esta manera se aplican los estilos en el mismo documento, en este caso letra roja 
 <h1 style='color:red;'>hola mundo</h1>

De esta manera se enlazan los estilos desde un archivo exterior, dentro de la etiqueta <head> 
 <head> <link rel="stylesheet" href="style.css"> <head>

Donde style.css es el nombre del archivo que contiene las clases para utilizarlas en el html Dentro del archivo style.css definimos una clase de la siguiente manera 
.text-red {
  color:red;
}

#text-red {
    color:red;
}

h1{
    color:blue;
}

Donde el . o # son el selector Selector { propiedades } y dentro de los corchetes se definen las propiedades como color tipografica tamaños etc, cade anotar que tambien las etiquetas se pueden usar como selectores directamente por ejemplo el h1 que se muestra en el ejemplo.


despues de importarlo para aplicar los estilos se hace de la siguiente manera

<h1 class=”text-red”>hola mundo</h1>
<h1 id=”text-red”>hola mundo</h1>

Como se puede observar se usan dos conceptos diferentes lo que son clases e id para la clase en css se utiliza el punto al inicio del nombre de la clase, Para el id se utiliza la # Los id o identificadores son únicos deben serlo, si hay id repetidos al igual que la clase tendrá efecto en todas las coincidencias pero la recomendación es los id son únicos, y solo se deben aplicar a un elemento mientras que las clases esas se pueden repetir para los selectores de etiqueta ejemplo. h1{ color:blue; } tambien tendra efecto en todos los h1, para el ejeplo uso h1 pero se recomienda que el h1 se utilize solo 1 vez por pagina el titulo.


Como decía al principio es Cascading Style Sheets traduce a hoja de estilo en cascada, tener esto en cuenta es muy importante porque por ejemplo importamos dos archivos, 

 <head> 
  <link rel="stylesheet" href="style.css"> 
  <link rel="stylesheet" href="reescribir.css">
<head> 

style.css
  .text-red {
  color:red;
}

reescribir.css
.text-red {
    color:blue;
}

como podemos ver se enlazan dos archivos css cada uno tiene dos clases con el mismo nombre cada una se encuentra en archivos diferentes pero tomara los estilos del ultimo archivo importado en este caso reescribir.css y su color sera blue azul.

hay 3 tipos de importancia o de dar prioridad, por ejemplo


  • 1. estilo inline o css directamente en html tiene mas prioridad que otros es el estilo que se aplica con por encima de los demas
    • <h1 style='color:red;'>hola mundo</h1>
  • 2. Estilos enlazados en la misma pagina tiene prioridad dos por encima que el que se elnaza desde otro archivo
    • <head>
     <style>
         h1  { color: red; }
     </style>
    <head>
  • 3. los archivos enlazados tienen la prioridad mas baja pero usando el important por ejemplo
    • .text-red{

¿Qué es un preprocesador de CSS?

Un preprocesador CSS es un programa que te permite generar CSS a partir de la syntax (en-US) única del preprocesador. Existen varios preprocesadores CSS de los cuales escoger, sin embargo la mayoría de preprocesadores CSS añadiran algunas características que no existen en CSS puro, como variable, mixins, selectores anidados, entre otros. Estas características hacen la estructura de CSS más legible y fácil de mantener.

CSS | MDN. (2021, 8 octubre). DSW-Andres-Meza. https://developer.mozilla.org/es/docs/Glossary/CSS_preprocessor

Un preprocesador de CSS se puede definir como una herramienta que nos permite escribir pseudocódigo CSS que luego será compilado de convertir en CSS tal y como lo conocemos de forma habitual.
Los preprocesadores nos permiten hacer cosas que con css puro no porque es como si nos diera la posibilidad de utilizar esta herramienta como si fuera un lenguaje de programación y luego la compila al css para mas información acerca de esto para mas informacion: Mozilla CSS_preprocessor
